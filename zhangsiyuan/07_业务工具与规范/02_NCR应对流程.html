公司治理 · Day 4 · Work 07

NCR 不符合项应对流程

Non-Conformance Report — 货代企业从问题发现到根因闭环的完整七步应对机制

什么是 NCR不符合已约定要求（合同/标准/法规）的事实或状态，包括货损、延误、单证错误、审核发现等。
为何需要闭环未闭环的 NCR 会在下次审核（ISO / 客户审核）中成为重复不符合，影响认证有效性和客户信任。
与 CAPA 的关系NCR 记录"发生了什么"；CAPA（纠正与预防措施）回答"为什么发生"和"如何防止再发"。
不符合项严重程度分级
Level 1 · Critical
严重不符合
  • 货物全损 / 严重货损
  • 海关违规 / 被处罚
  • 重大安全事故
  • ISO Major NC（认证风险）
  • 客户提出法律索赔
⚡ 响应：2 小时内
Level 2 · Major
重大不符合
  • 货物延误超 48 小时
  • 错误装机 / 目的港错误
  • 单证严重错误（AWB 收货人）
  • 客户正式投诉
  • 供应商重大违约
⏱ 响应：4 小时内
Level 3 · Minor
一般不符合
  • 轻微延误（12–48h）
  • 单证小错误（可更正）
  • 操作流程偏差
  • 内审发现轻微问题
  • 客户口头投诉
📋 响应：1 个工作日
Level 4 · OFI
改进机会
  • 流程可优化点
  • 效率提升建议
  • 风险预防性发现
  • 最佳实践推广建议
💡 响应：计划周期内
NCR 应对七步流程（点击每步查看详细操作）
01发现

发现与识别

Discovery & Identification — 不符合项触发条件
立即
▾

主动发现渠道

  • OPS 日常操作中发现的货物状态异常
  • 内部质量巡查 / 系统数据异常报警
  • 月度结算时发现的账单与实际不符
  • ISO 内部审核（Internal Audit）发现
  • 管理评审会议中识别的系统性问题

被动触发渠道

  • 客户发送的投诉邮件（Complaint Email）
  • 收货人 / 目的港代理的异常通知
  • 航空公司 / 船公司的货损通知单
  • 海关 / 检验检疫部门的处罚通知
  • 供应商申报的不可抗力或异常情况
关键：第一发现人责任。不论通过何种渠道发现 NCR，第一发现人必须在 30 分钟内通知直属上级，不得隐瞒、拖延或私自处理。"发现即上报"是 NCR 管理体系的基础原则。
02响应

即时响应与遏制

Immediate Containment — 防止损失扩大
2–4 小时
▾

遏制行动（按场景）

  • 货物延误：立即联系航司协查货物位置，安排替代航班，向客户发出延误通知（含原因+新 ETA）
  • 货损：第一时间拍照留证，通知航司开具货损报告（Damage Report），同步告知客户及保险公司
  • 单证错误：评估是否影响清关；若已起飞立即联系目的港代理；若未起飞立即申请 Amendment
  • 海关查验：协调查验场，通知客户可能延误，提前评估替代方案

升级路径

  • Level 1（严重）：立即报告总经理，财务评估损失，法务介入索赔准备
  • Level 2（重大）：部门负责人主导，OPS + 财务协同处理，24h 内向客户出具书面说明
  • Level 3（一般）：操作主管处理，业务员跟进客户，48h 内关闭
  • 任何客户正式投诉须在 2 小时内回复确认收到，4 小时内给出初步处理方案
遏制 ≠ 解决：遏制行动的目的是防止损失继续扩大（如：货物还能抢救则立即行动），而非等待根因分析后才采取行动。两者可以并行推进，不能以"等根因分析"为由延误遏制。
03记录

记录与分类

Documentation & Classification — NCR 表单填写
当日内
▾

NCR 表单必填项

  • 唯一编号（自动生成：NCR-YYYYMM-NNN）
  • 发现日期、发现人、发现渠道
  • 涉及案件号（HAWB/B/L No.）
  • 客户名称、货物品名、目的港
  • 事件详细描述（5W1H 格式）
  • 严重程度分级（L1 / L2 / L3 / OFI）
  • 责任部门 / 责任人
  • 直接经济损失估算

分类维度

  • 按业务环节：订舱 / 提货 / 报关 / 仓储 / 文件 / 结算
  • 按责任主体：自身操作失误 / 供应商失误 / 客户原因 / 不可抗力
  • 按影响维度：货物安全 / 时效 / 合规 / 财务 / 客户满意度
  • 按 ISO 条款：关联 ISO 9001:2015 相关条款编号（用于审核统计）
  • 分类准确是后续趋势分析的基础，切勿笼统归为"其他"
5W1H 描述模板：What（发生了什么）— Where（在哪个环节）— When（何时发现/发生）— Who（涉及哪些人员）— Which（涉及哪票货/哪个客户）— How（经过描述）。遵循此格式可让后续处理人员快速了解全貌，避免反复追问。
04分析

根本原因分析

Root Cause Analysis (RCA) — 找到真正的"为什么"
3–7 天
▾
RCA 核心原则：根本原因通常不是"人为失误"，而是导致人为失误的系统性原因（流程缺失、培训不足、工具不当、资源不够）。"操作员粗心"永远不是根本原因。
🔍 5-Why 分析法

连续追问"为什么"5次，层层深入直至找到根本原因。适用于原因链条清晰的单一失效场景。

Why1: AWB 收货人填错？→ 操作员录入错误
Why2: 为何录入错误？→ 未核对客户确认邮件
Why3: 为何未核对？→ 没有强制核对的 SOP
Why4: 为何没有 SOP？→ 流程文件未覆盖此场景
→ 根因：SOP 文件缺失
🐟 鱼骨图（石川图）

从人（Man）、机（Machine）、料（Material）、法（Method）、环（Environment）五个维度识别所有潜在原因。适用于复杂多因素问题。

货损案例：
人：操作员未确认包装规格
机：叉车操作不当
料：客户包装强度不足
法：堆码规则未执行
环：仓库湿度超标
📊 数据趋势分析

查阅过去 3–6 个月的 NCR 台账，分析是否为重复性问题，计算频率与损失金额，识别高风险环节。

如"延误"类 NCR 连续3月排第一，则说明根因在订舱或舱位管理环节，而非单次偶发。
🗣️ 访谈与现场观察

直接访谈一线操作人员，了解实际工作中的困难、习惯性做法和隐性障碍。现场观察实际操作流程。

通常会发现：书面 SOP 与实际操作之间存在差距，操作员有"非官方的应对方法"，往往才是根因的重要线索。
05措施

制定纠正与预防措施

Corrective & Preventive Action (CAPA)
7–14 天制定
▾

纠正措施（Corrective Action）

  • 针对已发生的不符合项：消除其直接原因（治标）
  • 修订相关 SOP / 作业指导书
  • 对涉事人员开展专项培训
  • 增加关键操作节点的双重核查（Four-eye Principle）
  • 在系统中添加强制校验或提醒逻辑
  • 调整供应商合同条款或更换供应商

预防措施（Preventive Action）

  • 针对潜在风险：防止类似问题在其他场景发生（治本）
  • 在风险评估（FMEA）中新增该风险点
  • 将本次 NCR 案例纳入新员工培训教材
  • 建立该类风险的预警指标和监控机制
  • 向同类客户/同类航线主动排查潜在风险
  • 与其他部门共享该 NCR 案例（横向推广）
SMART 原则：每条 CAPA 措施须符合 SMART 原则——Specific（具体描述做什么）、Measurable（如何衡量是否有效）、Assignable（指定唯一责任人）、Realistic（现有资源可实现）、Time-bound（明确完成截止日期）。
06验证

实施与有效性验证

Implementation & Verification
措施后 30 天
▾

实施跟踪

  • 品质管理部每周检查 CAPA 实施进度
  • 责任人在 NCR 台账中更新完成状态
  • 文件类措施：确认新 SOP 已正式发布并培训到人
  • 系统类措施：确认 IT 修改已上线并经过测试
  • 人员类措施：确认培训记录已存档

有效性验证方法

  • 同类操作抽样检查：随机抽取 10 票类似案件核查是否按新流程执行
  • 数据对比：CAPA 实施后同类 NCR 发生频率是否下降
  • 客户反馈：相关客户满意度是否改善
  • 内审复查：下次内部审核时专项审查该措施的执行情况
  • 若验证失败（问题复发），须重新进入 RCA 流程
验证 ≠ 措施完成：许多企业把"措施实施完成"等同于"NCR 关闭"，这是最常见的误区。只有经过客观验证、确认措施有效、问题不再复发，NCR 才可以正式关闭。ISO 审核员会重点核查验证记录。
07关闭

关闭与经验沉淀

Closure & Lessons Learned — 从个案到制度
验证通过后
▾

关闭条件

  • 所有纠正措施已完成并有记录
  • 有效性验证通过（数据或现场核查证明）
  • 受影响客户已收到最终处理结果说明
  • 如有财务赔偿，赔偿已完成并核销
  • 品质管理部负责人正式签署关闭确认

经验沉淀输出

  • 将典型 NCR 案例纳入内部知识库（脱敏处理）
  • 月度 NCR 趋势分析报告（频率 / 类型 / 损失金额）
  • 在管理评审会议上汇报月度 / 季度 NCR 统计
  • 更新公司风险评估文件（FMEA）
  • 高频或高损失 NCR 类型纳入下年度培训计划
重复 NCR = 体系缺陷：同一类型 NCR 在 12 个月内发生 3 次以上，即可认定为系统性缺陷而非个案。此时必须升级处理：由管理层主导专项改善项目，而非继续在个案层面处理。
